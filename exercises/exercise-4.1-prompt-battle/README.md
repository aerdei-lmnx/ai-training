# Exercise 4.1: Prompt Battle

## Goal

Learn that **prompt quality directly impacts output quality**. Two developers write prompts for the same task, run them in Claude Code, and the class votes on which output is better.

## Time

~20 minutes

## Format

1. **Pair up** with another developer.
2. Pick one of the coding tasks from `coding-tasks.md` (your instructor may assign one).
3. Each person independently writes a prompt for the same task.
4. Both run their prompts in Claude Code.
5. Compare outputs side by side.
6. The class votes on which output is better using the criteria in `scoring-rubric.md`.

## Rules

- You can **iterate on your prompt up to 3 times**. Your final result is what gets judged.
- You may include as much or as little context in your prompt as you like -- that is the whole point of the exercise.
- Keep track of each version of your prompt so you can discuss what you changed and why.

## Tips

- Think about what information Claude Code needs to produce a good result.
- Consider specifying things like edge cases, coding style, or output format.
- More specific prompts tend to produce better results, but verbosity alone does not help.

## Discussion (after voting)

- What made the winning prompt better?
- What context did it provide that the other prompt did not?
- Did either prompt constrain the output in a helpful way (e.g., "use type hints", "include docstrings")?
- How did iteration improve (or not improve) the results?
