# AI-Assisted Development Training Course

A practical, hands-on training course for developers learning to work effectively with AI coding assistants like Claude Code. This course teaches the mindset, techniques, and tools needed to get the best results from AI in real-world development work.

## Course Overview

This full-day training (6-8 hours) is designed for Python developers with varying experience levels. It covers:

- **Understanding Context** - Why AI assistants need good context and how to provide it
- **CLAUDE.md & Config Files** - Using persistent context to shape AI behavior
- **Prompt Engineering for Developers** - Writing better prompts to get better results
- **Building Your Toolkit** - Practical techniques for code review, debugging, and refactoring
- **Real-World Features** - Planning and implementing features with AI assistance

## Course Structure

### Slides
The main content is delivered through interactive reveal.js slides in the `slides/` directory. See [slides/README.md](slides/README.md) for setup and viewing instructions.

Each exercise has its own README with detailed instructions. Start with:
- `exercises/exercise-2.1-context-detective/README.md`
- `exercises/exercise-3.1-write-your-ai-briefing/README.md`
- And so on...

## Getting Started

### For Instructors

1. **Run the slides**: See [slides/README.md](slides/README.md)
2. **Set up exercises**: Each exercise is self-contained. Participants can work through them in order or jump to specific ones.
3. **Facilitator Notes**: Check individual exercise READMEs for timing, discussion questions, and tips.

### For Participants

1. Read the slides to learn concepts
2. Work through exercises 2.1 → 3.1 → 4.1 → 6.1 in order (they build on each other)
3. Try 4.2 and 7.1 as time allows
4. Discuss takeaways and share what worked

## Key Concepts

### Context is Everything
AI assistants start with a blank slate. The quality of help you get depends directly on the quality of context you provide.

### CLAUDE.md Files
Project-specific briefing files that teach AI about your conventions, architecture, and standards. More effective than lengthy documentation.

### Prompt Quality Matters
Small improvements to prompts create dramatic improvements in output. Specificity, clarity, and examples all matter.

### Effective Iteration
Plan → Ask → Review → Refine. Working with AI is a conversation, not a one-shot request.
